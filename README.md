# feed-mysql-bridge

[English version](README.en.md)

Мост, который переносит спортивный дата-фид в вашу базу MySQL и поддерживает её
в актуальном состоянии в реальном времени.

Фид состоит из двух частей:

- **Redis** — полное текущее состояние линии (снимок). Используется для начальной
  загрузки (cold start).
- **RabbitMQ** — поток инкрементальных обновлений (AMQP) поверх снимка.

`feed-mysql-bridge` сначала загружает снимок из Redis в MySQL, а затем слушает
очередь RabbitMQ и применяет обновления по мере поступления.

---

## Требования

- Python 3.7+
- Библиотеки: `pip install redis PyMySQL pika`
- Доступная база MySQL/MariaDB

---

## Установка

1. **Создайте базу и таблицы.** Скрипт `schema.sql` сам создаёт БД `feed_bridge`
   (`CREATE DATABASE IF NOT EXISTS feed_bridge`) и все таблицы. Выполните под
   пользователем MySQL с правом создавать базы:
   ```
   mysql -u root -p < schema.sql
   ```

2. **Создайте пользователя MySQL для моста** и выдайте ему права на `feed_bridge`
   (либо используйте существующего пользователя с доступом к этой базе):
   ```sql
   CREATE USER 'feed_bridge'@'127.0.0.1' IDENTIFIED BY 'ВАШ_ПАРОЛЬ';
   GRANT SELECT, INSERT, UPDATE, DELETE ON feed_bridge.* TO 'feed_bridge'@'127.0.0.1';
   FLUSH PRIVILEGES;
   ```

3. **Заполните `config.json`.** В этом репозитории `config.json` уже содержит
   выданные вам **доступы к фиду** (`redis`, `rabbitmq` и вашу персональную очередь
   `rabbitmq.queue`) — их менять не нужно. Впишите только секцию `mysql`: замените
   плейсхолдеры `<mysql-user>` / `<mysql-password>` на пользователя и пароль из шага 2
   (`host`, `port` и `database: feed_bridge` уже проставлены):
   ```json
   "mysql": { "host": "127.0.0.1", "port": 3306,
              "user": "feed_bridge", "password": "ВАШ_ПАРОЛЬ",
              "database": "feed_bridge", "charset": "utf8mb4" }
   ```
   > Полный шаблон всех полей — в `config.example.json`. Секции `redis`/`rabbitmq` —
   > это доступы к фиду от оператора; в вашей копии они уже заполнены.

## Запуск

```
python bridge.py              # снимок, затем слушать поток (по умолчанию)
python bridge.py --snapshot   # только снимок и выход
python bridge.py --listen     # без снимка, сразу слушать поток
```

При старте без флага (или с `--snapshot`) мост очищает очередь RabbitMQ и таблицы,
затем заново загружает всё из Redis. Режим `--listen` применяет обновления поверх
уже загруженной базы.

Для постоянной работы (автозапуск при загрузке, автоперезапуск при сбое) мост
оформляется как **systemd-сервис в режиме `--listen`** — он не делает снапшот и не
чистит БД при рестарте, поэтому не теряет результаты исходов, пришедшие по AMQP.
Юнит и инструкция: [`deploy/feed-bridge.service`](deploy/feed-bridge.service) и
[`DEPLOYMENT.md`](DEPLOYMENT.md).

python db_info.py             # размеры таблиц и разбивка строк по статусам

## Структура

| Путь | Назначение |
|------|------------|
| `bridge.py` | Точка входа: разбор аргументов, подключения, запуск режимов. |
| `feedbridge/` | Пакет с логикой моста (см. ниже). |
| `db_info.py` | Утилита просмотра состояния базы. |
| `schema.sql` | DDL таблиц фида. |
| `config.example.json` | Шаблон конфигурации. |

Пакет `feedbridge/`:

| Модуль | Назначение |
|--------|------------|
| `config` | Константы и загрузка `config.json`. |
| `console` | Цвета и форматирование вывода. |
| `transform` | Конвертеры значений Redis-JSON → MySQL. |
| `sql` | SQL-операторы upsert с защитой версией. |
| `db` | Подключение к MySQL. |
| `amqp` | Подключение к RabbitMQ, глубина и очистка очереди. |
| `core` | `Bridge`: соединения, кэши идентификаторов, разрешение ID. |
| `snapshot` | `Snapshotter`: полная загрузка Redis → MySQL. |
| `listener` | `Listener`: инкрементальные обновления AMQP → MySQL. |

Каждый модуль начинается с подробного описания назначения и функций.
